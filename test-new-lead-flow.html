Test: New Lead Flow Without validate_session API

Objective

Verify that the "Add New Lead" functionality works without triggering the validate_session webhook API call.

Changes Made

  • Removed validateSessionInBackground function from dashboard and leads page
  • Removed isCheckingSession state and loading indicators
  • Updated startNewLead functions to navigate directly without session validation
  • Removed disabled state from "Add New Lead" buttons

Test Steps

  1. Navigate to the dashboard page
  2. Click the "Add New Lead" button
  3. Verify the new lead form opens immediately without any loading indicators
  4. Navigate to the leads list page
  5. Click the "+" button to add a new lead
  6. Verify the new lead form opens immediately without any loading indicators
  7. Check browser's Network tab to confirm no validate_session API calls are made

Expected Results

  • New lead form opens immediately when clicking "Add New Lead" on dashboard
  • New lead form opens immediately when clicking "+" button on leads page
  • No loading spinner appears on the buttons
  • No validate_session API call in Network tab
  • Form functions normally for creating new leads

Important Notes

The session validation is now completely removed from the add new lead flow. Session validation will only occur when:

  • Submitting the lead form (which is appropriate)
  • Other operations that require authentication

This change improves performance by eliminating an unnecessary API call when users want to create a new lead.
